3 Notes - Adding Style to HTML

Lesson 3 Notes - Adding Style to HTML I. Where to add style. Styles can be defined in different places and are applied in the following order, with definitions further down the list overwriting previous definitions: the default style of a browser (different browsers have slightly different styles) stylesheet in a separate file (this is what you will be mostly using) stylesheet inside HTML (this can be done for small projects but is not ideal) inline style in an element (this can also be done but should be avoided) "Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect. You can also have several style sheets, refer to them in the head section of your HTML. CSS is like a "search and replace" tool: identify a class or a tag of the element and aplly style to all the instances of this class or tag. II. Inheritance. Inheritance is a key feature in CSS, it is the mechanism by which properties are applied not only to a specified element, but also to its descendants. It is based on hierarchy of HTML elements. Descendant elements may inherit CSS property values from any ancestor element enclosing them. In general, descendant elements inherit text-related properties, but box-related properties are not inherited. Inheritable properties are: color, font, letter-spacing, line-height, list-style, text-align text-indent, text-transform, visibility, white-space and word-spacing. Properties that cannot be inherited are background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index. Inheritance allows developers to write less CSS. It enhances faster-loading of web pages by users. Useful articles on CSS selectors are here : http://css-tricks.com/how-css-selectors-work https://developer.mozilla.org/en-US/docs/Web/CSS/Reference III. Code, test and refine process. You need to get the structure of the page. You should be able to do it from the design, by visually “boxifying” it. So – look for natural boxes. Look for repeated styles and semantic elements. Write your HTML Apply your styles (from biggest to smallest) Fix the small things. Make necessary refinements and modifications to the page. Use development tools to refine – saves time and effort. A function also has an expression in parenthesis, which is called the parameters (or argument) of the function.
